# 局域网文件消息传输工具 - 设计规格

**日期**: 2026-06-29  
**工具ID**: `lanshare`  
**工具名称**: 局域网快传

---

## 1. 概述

纯前端实现的局域网内消息与文件传输工具，基于 WebRTC DataChannel 实现点对点传输，无需服务器，运行在 ToolBox WebView 环境中。

### 设计目标
- ✅ 纯前端实现，无后端依赖
- ✅ 支持文本消息传输
- ✅ 支持大文件传输（浏览器内存上限）
- ✅ 对等连接模式
- ✅ 消息与连接状态持久化

---

## 2. 技术选型

| 技术 | 用途 | 说明 |
|------|------|------|
| WebRTC RTCPeerConnection | 点对点连接建立 | 标准浏览器 API |
| RTCDataChannel | 数据传输通道 | 可靠有序传输 |
| 手动信令 | 连接信息交换 | 用户复制粘贴 SDP/ICE |
| localStorage | 数据持久化 | 消息历史 + 连接配置 |
| FileReader API | 文件读取分片 | 大文件流式处理 |

## 4. 核心功能设计

### 4.1 连接管理

#### 连接建立流程
```
设备A                          设备B
  │                              │
  ├─ 创建 PeerConnection        │
  ├─ 创建 DataChannel           │
  ├─ 生成 Offer SDP             │
  ├─ 显示 Offer 给用户           │
  │                              │
  └───────── 用户复制 Offer ────▶│
  │                              │
  │                              ├─ 粘贴 Offer
  │                              ├─ 创建 PeerConnection
  │                              ├─ setRemoteDescription
  │                              ├─ 生成 Answer SDP
  │                              ├─ 显示 Answer 给用户
  │                              │
  ◀──────── 用户复制 Answer ─────┘
  │
  ├─ setRemoteDescription(Answer)
  │
  ├─ 交换 ICE Candidate (手动或自动)
  │
  └─ 连接建立
```

#### 连接状态
- `new` - 新建
- `connecting` - 连接中
- `connected` - 已连接 ✓
- `disconnected` - 已断开
- `failed` - 失败
- `closed` - 已关闭

#### 持久化
- 保存最近 5 个连接配置
- 保存连接的设备名称、公钥指纹
- 刷新页面后自动列出历史连接

### 4.2 消息传输

#### 消息格式
```javascript
{
  type: "text" | "file" | "file-chunk" | "ack",
  id: "uuid-string",
  from: "device-id",
  timestamp: 1234567890,
  // text 类型
  content: "消息内容",
  // file 类型（首包）
  fileId: "file-uuid",
  fileName: "document.pdf",
  fileSize: 2097152,
  fileType: "application/pdf",
  // file-chunk 类型
  chunkIndex: 5,
  totalChunks: 32,
  data: "base64-encoded-chunk"
}
```

#### 消息状态
- 发送中 → 已送达（收到 ack）

### 4.3 文件传输

#### 分片策略
- 初始分片大小：64KB
- 根据传输延迟动态调整（32KB - 256KB）
- 每分片发送后等待确认（滑动窗口可后续优化）

#### 传输流程
```
发送方                        接收方
  │                             │
  ├─ 选择文件                   │
  ├─ 计算总分片数               │
  ├─ 发送文件元信息包           │
  │                             │
  ├─ 发送分片 0 ──────────────▶│
  ◀──────────── ACK 0 ─────────┤
  ├─ 发送分片 1 ──────────────▶│
  ◀──────────── ACK 1 ─────────┤
  │  ...                        │
  │                             │
  └─ 最后分片 → 传输完成        │
```

#### 文件大小限制
- 理论上限：浏览器可用内存（通常 500MB - 2GB）
- 实际推荐：≤ 500MB（避免内存压力）

## 7. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 浏览器不支持 WebRTC | 显示友好提示，指导用户升级浏览器 |
| 连接建立失败 | 显示具体错误 + 重试按钮 |
| 传输中断 | 保存已接收分片，重连后询问是否续传 |
| 文件过大 | 选择文件时即时提示，拒绝添加 |
| 内存不足 | 捕获异常，提示用户分批发送 |

---

## 8. 边界情况

- ✅ 空消息：禁用发送按钮
- ✅ 零字节文件：允许发送但提示用户
- ✅ 并发传输：队列化，同一时间只传一个文件
- ✅ 页面刷新：消息历史保留，连接状态重置
